# 🧪 TEST PLAN

## 1. Introduction  
Bu test planın məqsədi tətbiqin funksionallığını yoxlamaq, səhvləri tapmaq və məhsulun keyfiyyətini təmin etməkdir.

## 2. Scope  
Test ediləcək sistem: **[Sistemin adı]**  

Test növləri:
- Functional testing  
- Regression testing  
- UI testing  
- API testing (əgər varsa)

## 3. Objectives  
- Sistemin düzgün işlədiyini yoxlamaq  
- Bug-ları aşkar etmək  
- İstifadəçi tələblərinə uyğunluğu təmin etmək  

## 4. Test Items  
Test olunacaq modullar:
- Login / Registration  
- Dashboard  
- User Profile  
- Payment (əgər varsa)  
- API endpointlər  

## 5. Features to be Tested  
- Düzgün login/logout  
- Məlumatların düzgün saxlanması  
- UI elementlərin işləməsi  
- API response-ların doğruluğu  

## 6. Features NOT to be Tested  
- 3rd party servisler (məs: payment gateway)  
- Test mühitindən kənar sistemlər  

## 7. Test Strategy  
Testlər aşağıdakı üsullarla aparılacaq:
- Manual testing  
- Black-box testing  
- Positive & Negative testing  
- Exploratory testing  

## 8. Test Environment  
- OS: Windows 10 / 11  
- Browser: Chrome, Firefox  
- Toollar: Postman, Jira  
- Test server: staging environment  

## 9. Entry Criteria  
- Requirements hazırdır  
- Build deploy olunub  
- Test mühiti işləkdir  

## 10. Exit Criteria  
- Critical bug yoxdur  
- Test case-lərin 95%-i pass olub  
- Report hazırlanıb  

## 11. Test Deliverables  
- Test Plan  
- Test Cases  
- Bug Reports  
- Test Summary Report  

## 12. Roles and Responsibilities  
- QA Engineer – testlərin icrası  
- Developer – bug fix  
- Project Manager – nəzarət  

## 13. Risks and Mitigation  
Risklər:
- Gec build gəlməsi  
- Requirement dəyişməsi  

Həll:
- Agile işləmək  
- Continuous testing  

## 🔥 CV üçün istifadə
Created and executed test plans, test cases, and test scenarios to ensure software quality. Performed functional, regression, and API testing, and reported defects using bug tracking tools.
